🎯 섹션별 목적

Projects (100)

현재 진행 중인 활동적 프로젝트와 명확한 목표가 있는 작업들을 포함합니다. 특정 기간 내에 완료되어야 하는 작업들이 여기에 속합니다.

Areas (200)

지속적인 관리와 관심이 필요한 영역입니다. 장기적인 책임과 역할, 지식 관리가 이루어지는 공간입니다.

Resources (300)

참고자료, 학습자료, 도구 등 나중에 참조할 수 있는 모든 자료들이 포함됩니다.

Archive (900)

완료된 프로젝트와 더 이상 활성화되지 않는 자료들을 보관하는 공간입니다.

Inbox (000)

새로운 노트의 임시 저장소로, 나중에 적절한 위치로 분류될 내용들을 임시로 보관합니다.

Templates (800)

재사용 가능한 노트 템플릿들을 보관하는 공간입니다.

📊 폴더 특성

100_Projects/

  • ⏱️ 시간 제한이 있는 활동적 프로젝트
  • 🎯 명확한 목표와 데드라인 존재
  • 📊 진행 상황 추적 가능
  • ✅ 완료 여부가 명확함

200_Areas/

  • 🔄 지속적인 관리가 필요한 영역
  • 📚 장기적인 지식 축적
  • 🎯 명확한 완료 시점이 없음
  • 💡 꾸준한 개선과 발전이 필요한 분야

300_Resources/

  • 📚 참고자료 및 학습자료
  • 🔍 주제별 정보 저장소
  • 🛠️ 도구 및 템플릿
  • 📖 나중에 참조할 자료들

900_Archive/

  • 📦 비활성 프로젝트 저장
  • 🗄️ 완료된 프로젝트
  • 📝 과거 기록 보관
  • 🔍 필요시 참조 가능한 형태로 유지

000_Inbox/

  • 📥 새로운 아이디어 임시 저장
  • 🔄 정기적인 정리 필요
  • 📝 빠른 캡처 위주
  • ⚡ 즉각적인 기록 가능

800_Templates/

  • 📋 재사용 가능한 템플릿
  • 🔄 표준화된 형식
  • ⚡ 빠른 문서 생성
  • 📝 일관된 포맷 유지

💡 구현을 위한 팁
  • 새로운 노트는 항상 Inbox에 먼저 작성
  • 주기적인 정리를 통해 적절한 위치로 이동
  • 완료된 프로젝트는 즉시 Archive로 이동
  • Templates 활용으로 일관된 노트 형식 유지
  • 태그 시스템과 함께 사용하여 크로스 레퍼런싱 강화
